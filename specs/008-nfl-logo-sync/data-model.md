# Data Model: NFL Team Logo Sync & Draft Room Display

> Generated by `/speckit.plan` (Phase 1).

---

## Entities

### NflTeam

Represents a single NFL franchise. Persisted in the `nfl_teams` table in
`packages/backend/data/capstone.db`. Exposed to the frontend via `GET /api/nfl-teams`.

| Field | Type | Constraints | Notes |
|-------|------|-------------|-------|
| `id` | INTEGER | PRIMARY KEY AUTOINCREMENT | Internal surrogate key |
| `espn_team_id` | TEXT | NOT NULL, UNIQUE | ESPN's numeric ID (e.g. `"16"`) |
| `name` | TEXT | NOT NULL | Team name (e.g. `"Vikings"`) |
| `location` | TEXT | NOT NULL | City/location (e.g. `"Minnesota"`) |
| `abbreviation` | TEXT | NOT NULL, UNIQUE | Uppercase abbreviation (e.g. `"MIN"`) |
| `color` | TEXT | nullable | Primary hex color without `#` (e.g. `"4f2683"`) |
| `alternate_color` | TEXT | nullable | Alternate hex color without `#` |
| `logo_path` | TEXT | nullable | Root-relative browser path `/logos/nfl/min.png`; `NULL` if sync has not run |
| `created_at` | TEXT | NOT NULL, DEFAULT `datetime('now')` | ISO-8601 UTC |
| `updated_at` | TEXT | NOT NULL, DEFAULT `datetime('now')` | ISO-8601 UTC; updated on upsert |

**Schema DDL** (created on first `nflTeamsStore` import):

```sql
CREATE TABLE IF NOT EXISTS nfl_teams (
  id               INTEGER PRIMARY KEY AUTOINCREMENT,
  espn_team_id     TEXT NOT NULL UNIQUE,
  name             TEXT NOT NULL,
  location         TEXT NOT NULL,
  abbreviation     TEXT NOT NULL UNIQUE,
  color            TEXT,
  alternate_color  TEXT,
  logo_path        TEXT,
  created_at       TEXT NOT NULL DEFAULT (datetime('now')),
  updated_at       TEXT NOT NULL DEFAULT (datetime('now'))
);
```

**Upsert strategy** (sync script): `INSERT OR REPLACE INTO nfl_teams (espn_team_id, name, location, abbreviation, color, alternate_color, logo_path, updated_at) VALUES (...)` — overwrites on `espn_team_id` conflict so re-running sync updates existing rows.

### LogoFile

Not a database entity — represented as a file on disk.

| Attribute | Value |
|-----------|-------|
| Directory | `packages/frontend/public/logos/nfl/` |
| Filename pattern | `<lowercase_abbreviation>.png` (e.g. `min.png`) |
| Browser URL | `/logos/nfl/min.png` (root-relative static asset) |
| Written by | `packages/backend/scripts/syncNflLogos.js` |
| Served by | Create React App static file serving — no backend route |

## API Response Shape

`GET /api/nfl-teams` returns an array of team objects. Each object maps `nfl_teams` DB columns to camelCase frontend fields:

```json
[
  {
    "id": 1,
    "espnTeamId": "16",
    "name": "Vikings",
    "location": "Minnesota",
    "abbreviation": "MIN",
    "color": "4f2683",
    "alternateColor": "ffc62f",
    "logoPath": "/logos/nfl/min.png"
  }
]
```

`logoPath` is `null` when the sync has not been run for that team. The frontend must treat `null` as "no logo available" and show the abbreviation text fallback.

---

## Frontend State Shape

`DraftRoomPage` derives a lookup map from the API response:

```js
// Map<string, string | null>  abbreviation → logoPath
const nflTeamLogoMap = new Map(
  teams.map(t => [t.abbreviation, t.logoPath])
);
```

`DraftPlayerRow` receives `logoPath` (looked up by `player.nflTeamAbbr`) as a prop.
`DraftFilters` receives the full `nflTeamLogoMap` for rendering thumbnails in the filter.

---

## Relationships

```
NflTeam (1) ──────────────── (0..1) LogoFile
             espn_team_id           filename = abbreviation.toLowerCase() + '.png'
```

Players in the Draft Room reference NFL teams by abbreviation (`player.nflTeamAbbr`). The `nfl_teams` table is not foreign-keyed to the players data store (players remain in-memory constants); the join happens in the frontend via the `nflTeamLogoMap`.

---

## Validation Rules

| Field | Rule |
|-------|------|
| `espn_team_id` | Non-empty string from ESPN API response |
| `abbreviation` | Non-empty string; stored as-received from ESPN (uppercase); filename uses `.toLowerCase()` |
| `logo_path` | Set to `/logos/nfl/<abbr>.png` after successful download; `NULL` if download failed |
| `color` / `alternate_color` | Stored as-received (6-char hex without `#`); no validation — informational only |

---

## State Transitions (Sync Script)

```
Team record: absent
       ↓  sync runs, API fetch succeeds, logo download succeeds
Team record: present, logo_path = '/logos/nfl/<abbr>.png'

Team record: absent
       ↓  sync runs, API fetch succeeds, logo download fails
Team record: present, logo_path = NULL  (failure reported in summary)

Team record: present (from prior sync)
       ↓  sync re-runs, logo download succeeds
Team record: updated, logo_path refreshed, updated_at updated
```
